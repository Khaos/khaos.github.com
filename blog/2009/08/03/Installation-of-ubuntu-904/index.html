相对于安装Windows XP，安装Ubuntu9.04是个颇为惬意的事情。本文将介绍如何从Live CD安装Windows XP (SP3) 及 Ubuntu 9.04 (Jaunty Jackalope) 双系统的过程。文中假定你的电脑上已经安装了Windows XP，并且安装至硬盘的第一个主分区 (hda1/sda1, 分别对应与IDE/SCSI硬盘) 上，在Windows下，这个盘符一般为C:。

安装前准备

首先你需要下载Ubuntu，最新版9.04可以到这里下载。由于Ubuntu是Linux一个free的发行版本，所以不用担心版权问题，你甚至可以在那里申请免费的Live CD。在下载页面中，你可以根据所需选择需要的版本，例如如果你的机器配置比较好，64位的CPU，那么你可以选择64位Desktop版本，这样你就可以安装64位的Matlab，而远离 OUT OF MEMORY error 了。

Ubuntu的安装模式相对较多样化且贴心，例如你可以申请Live CD，而光盘安装；你也可以下载安装镜像文件，而在Windows下挂载镜像安装；你还可以只下载最小安装文件，在线飞行安装。

另一个准备工作就是给你的硬盘分好区，其实这一步在安装Ubuntu的时候也可以做，不过这里建议你先把Linux所用的分区划分出来，下面安装的时候就方便多了。我的分区方案是先划分成两个主引导分区 (sda1, sda2) 和一个扩展分区，然后再从扩展分区起始部分分出2个逻辑分区 (sda5, sda6，注意逻辑分区的分区号是从5开始），这些分区都是给Windows用的，这样在扩展分区的最后部分就余下大概50G的空间供Ubuntu使用。

安装过程

上面提到了分区，其实这种方式安装Ubuntu是“纯净”安装，也就是Windows和Ubuntu都在各自的分区上，互不干扰。从Ubuntu 8.10 （也可能是 8.04 alpha） 版本，它就自带一种在Windows下“嵌入式”的安装模式，Wubi。这种模式的优点是你可以像安装一个Windows标准软件一样安装与卸载Ubuntu，当然这种安装产生的也是原汁原味的Ubuntu系统，而非虚拟机式的；缺点就是在磁盘存取速度上会有所损失。所以如果你只是想尝鲜，而离不开Windows的怀抱，可以选择这种方式，详细介绍可参见上面给出的Wubi链接。由于我要把Ubuntu作为主力系统，所以采用建立独立分区的方式。

准备工作完成后，就可以安装Ubuntu鸟～～～

  • 把Live CD放入光驱，然后开机，从光盘启动。在启动界面上，会看到安装选项，其中一项是从CD运行Ubuntu，若想先过把瘾，可以选择它，试用一下。现在回到安装上来，选择安装Ubuntu，进入安装界面。
    1. 择安装语言，就是安装时，显示提示的语言，选 [English]。
    2. 择你的所在地，这个选项类似于Windows里的区域选项，可能会决定系统以何种默认编码读取你文档，以及设定相应的一些本地设置，例如货币单位的表示，选 [London]
    3. 择键盘类型，如果你是UK的键盘，就要选 [U. K. English]。

    4. 第四步，硬盘分区，这也是最关键的一步。一般来说，安装一个Ubuntu系统，至少需要两个分区，分别为 / 分区和 SWAP 分区，前者用于存放所有文件，而后者类似于Windows下的虚拟内存，一般建议大小为内存的 1.5~2 倍。如果想将用户文件单独存放，这就需要从 / 中分出一个 /home 分区；如果想把应用软件安装于独立的分区上，就需要分出一个 /usr 分区，以便于管理；另外，还可以分出一个 /var/log 分区，也就是系统日志记录分区，一个 /tmp 分区，用来存放临时文件，一个 /bin 分区，用来存放标准系统实用程序，等等。对于新手来说，一般分三个区就可以了，依次为 / ，/home，及SWAP。其中前两个分区的文件格式选择ext3（下一个版本的Ubuntu，ext4将成为默认文件格式），SWAP就是swap格式。下面是我的分区情况：

      / ext3 27G  
      /home ext3 20G  
      SWAP swap 3G 一个较大的swap，是因为一些数值运算软件，如Matlab，在处理大型数据时所需
    5. Migrate Documents and Settings，就是把你之前的一些用户信息（可以是Windwos上的）同步过来。
    6. 填写用户个人信息。
    7. 检查设定。
    8. 开始安装。
  • 最后，重启，完成。

一些说明

虽然Ubuntu安装光盘上包含了大多数硬件的驱动，但是还是强烈建议在有条件的情况下，连上有线网安装，这样在安装过程中，会自动下载最新的驱动程序进行安装。不仅如此，对于一些受限的硬件来说，例如无线网卡和Modem，必须手动激活并且在线下载驱动安装，所以在不接驳网络的情况下，很有可能找不到这两个硬件，而那个激活界面也不会自动出现。

总而言之，Ubuntu的安装还是相当轻快的，基本上不需要手动安装硬件驱动，有了GTK+，界面也是相当的人性化。

后记：
